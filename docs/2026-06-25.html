WorkOS推出auth.md规范：让AI代理无需注册表单即可认证用户

📝 daringfireball.net · Daring Fireball Department of Commerce 🕐 2026-06-24 18:46 AI
WorkOS 推出了名为 auth.md 的新规范，旨在让 AI 代理无需用户填写注册表单即可完成身份认证。该规范要求应用在其域名下托管一个 Markdown 文件，定义支持的认证流程、权限范围以及代理如何代表用户注册。目前该规范支持两种流程：Agent Verified（代理验证）和 User Claimed（用户认领），已被部分应用采用。 核心观点是：auth.md 通过标准化的 OAuth 机制，让 AI 代理能够代表用户安全地完成应用注册，而无需传统的人工填写表单环节。其中 Agent Verified 流程依赖代理的身份提供商进行无人工干预的用户认证，User Claimed 流程则通过展示验证码让用户手动确认。两种流程均颁发短期、可撤销的 scoped access token。 对从业者的启发是：auth.md 为 AI 代理与现有应用的集成提供了标准化接口，开发者应关注如何在自己的应用中实现该规范，以支持代理驱动的用户注册场景。同时需权衡不同认证流程的安全性与用户体验，例如 Agent Verified 适合自动化场景，User Claimed 则保留人工确认环节。该规范可能推动更多应用采用类似 OpenAI 的 GPT Actions 或 Anthropic 的 MCP 等代理协议。

文章指出，WebKit 作为苹果平台的核心浏览器引擎，不仅用于 Safari 显示网页，还用于 Mail 应用渲染邮件（包括纯文本格式），并揭示了两个可能属于 Bug 的表现：在辅助功能中禁用“自动播放动画”后，Mail 的右键菜单会异常出现“播放所有动画”选项；此外，无论是否有选中内容，WebKit 的 WebView 始终启用主菜单中的“复制”项，该问题在 Safari、Mail 及第三方 RSS 阅读器（如 NetNewsWire 和 Vienna）中均可复现。 核心观点是：WebKit 在 macOS Sequoia 上存在一个普遍性缺陷，即其 WebView 不会根据实际选中状态动态禁用“复制”菜单项，导致用户即使未选择任何内容也能触发复制操作（可能复制空白或意外内容）。这一行为源于 WebKit 内部对菜单状态管理的逻辑缺失，且影响所有依赖 WebKit 渲染内容的苹果原生及第三方应用。 对从业者的启发是：若开发基于 WebKit 的 macOS/iOS 应用（如 RSS 阅读器、邮件客户端），应主动检测并修复“复制”菜单项的启用逻辑，可通过监听 selectionchange 事件或调用 WebKit 的 canCopy 方法动态控制菜单状态；同时需警惕 WebKit 的辅助功能相关 Bug（如动画播放菜单异常），建议在系统设置中测试并添加针对性兼容代码，避免用户混淆。
